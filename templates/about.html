ABOUT THIS PROJECT

API - Formula 1 2025

Project Description

This web application is built as part of the Groupie Tracker project for Ynov. It provides a comprehensive interface to explore Formula 1 drivers and teams for the 2025 season.

Features

Drivers Management

Browse all F1 drivers with advanced filtering by team, nationality, and driver type. Pagination system for optimal navigation.

Teams Overview

Explore all F1 teams with detailed information about each constructor and their drivers.

Favorites System

Add your favorite drivers and teams to a persistent favorites list stored in JSON format.

Search Functionality

Quickly find drivers and teams using the search feature.

Technical Stack

Backend

  • Golang
  • REST API consumption (Ergast F1 API)
  • JSON file
  • Error handling with HTTP status codes

Frontend

  • HTML - Semantic markup
  • CSS - without frameworks
  • Go Templates - Server-side rendering
  • Responsive design

Constraints & Requirements

  • REST API exploitation for data retrieval
  • Complete error management (offline services, HTTP codes, error pages)
  • Only Golang, HTML, and CSS languages
  • Go standard libraries only
  • No HTML/JS/CSS frameworks
  • User-friendly interface
  • Filter system (3 cumulative filters)
  • Pagination system (10/20/30 items per page)
  • Favorites system with persistence

Data Source

This application uses the Ergast F1 API to retrieve Formula 1 data including drivers, teams, and season information.

Project Breakdown:

  • Phase 1 - Analysis & Setup (Class 1)

  • Phase 2 - Backend Core (Class 2-3)
    • Development of controllers (Drivers, Teams, Search)
    • Implementation of services (filtering, pagination, search)

    • Models and data structures
  • Phase 3 - Frontend & Templates (Class 4)
    • Creation of HTML templates
    • Custom CSS design (responsive, F1 fonts)
    • Favorites system
    • Audio persistence and player interface
  • Phase 4 - Advanced Features (Class 5)
    • Multiple filters (team, nationality, driver type)
    • Dynamic pagination
    • Unified global search
    • Immersive audio effects
  • Phase 5 - Polish & Testing (Class 6-7)
    • Browser compatibility tests
    • Performance optimizations
    • Documentation and README
    • Implementation of constraints
